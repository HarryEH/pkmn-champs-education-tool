## 1. Forme-family collapse (the Rotom-Wash miss)

**Problem.** The Rotom-Wash crop's signal was *diluted across six rotom appliance formes*: no single
forme ranked high (rotomwash sat at rank ~7, behind grey-ghost distractors cofagrigus/garganacl),
and surfacing one specific appliance as a chip was actively misleading — the appliances are visually
inseparable at team-preview render scale but **mechanically distinct** (Fan = Electric/Flying vs
Wash = Electric/Water).

**Fix.** Pool forme-families to one **base-species** candidate.
- `boxEmbeddings.json` entries now carry `baseSpeciesId` (emitted by `buildBoxEmbeddings.ts` via
  `Dex`; the committed table was migrated in place — no re-embedding).
- `matchEmbedding` gained a `collapseFormes` option: after ranking it dedupes best-per-`speciesId`,
  then (when collapsing) best-per-`baseSpeciesId`, surfacing the **base id** (`rotom`).
- **Only the user-facing chip list collapses.** Auto-accept still reasons over the *raw*
  specific-forme ranking — collapsing there would inflate the top1−top2 margin into wrong-forme
  auto-accepts.

**Result.** Base `rotom` now appears in the collapsed top-3 (chip reads "Rotom"; the user picks the
appliance from the existing full-species dropdown). Species-exact raw top-1 **unchanged at 5/6**, no
other slot affected. New assertion in `detectionAccuracy.test.ts` locks **base top-3 = 6/6**.

**Honest framing.** This is a *presentation* fix, not a recognition fix. The root cause of the hard
misses is that **every reference is a synthetic pokesprite box sprite** (domain gap to real Switch
renders) tuned against **one** labelled frame — which §3 attacks directly.

Files: `iconMatcher.ts`, `boxEmbeddings.ts`, `detectionPipeline.ts`, `scripts/buildBoxEmbeddings.ts`,
`src/data/boxEmbeddings.json`, `detectionAccuracy.test.ts`, `boxEmbeddings.test.ts`.

## 2. Multi-source Detection input

Detection's "Add opponent" card now has a **Screenshot / PokePaste / Video** tab switcher.

### 2.1 PokePaste → confirmed team + exact sets
- `pasteOpponent.ts` (pure, tested): parses an opponent Showdown export (reusing the team store's
  `parsePokepaste`) into a fully-confirmed `OpponentTeam` (`confidence: 1`, slot item/ability/Tera
  seeded) plus a `speciesId → PokemonSet` map. No CLIP needed.
- **Exact sets thread into calc.** New in-memory session field `opponentSets` (cleared on
  detection/"New Battle", never persisted — same policy as `opponent`). New `opponentBuild` helpers
  `opponentCombatant` / `opponentMoves` / `opponentSpeedInput` prefer the exact set; `buildMatrixCell`
  and `OpponentDashboard` use the real set/EVs/moves/speed and skip the usage-variant picker. The
  matrix column header + cells stay consistent with the set.
- `InBattle` deliberately keeps its **unknown-EV speed-range** model, but still picks up the revealed
  item/Tera from the slot.

### 2.2 Video (scaffold)
- `videoSource.ts::grabVideoFrame` captures a `<video>` still as the canonical `RgbaImage` + a PNG
  snapshot URL — the same shape the screenshot path produces, so the CLIP pipeline runs unchanged.
- `VideoCapture.tsx`: device picker (persists `captureDeviceId`) + live preview + "Grab still &
  detect" → existing calibrate→Detect flow. **Hardware validation deferred** to the Elgato session.

Files: `pasteOpponent.ts`, `PasteInput.tsx`, `VideoCapture.tsx`, `videoSource.ts`,
`store/session.ts`, `opponentBuild.ts`, `matrixBuild.ts`, `MatchupMatrix.tsx`, `OpponentDashboard.tsx`,
`Detection/index.tsx`; `team-pastes/` sample pastes for testing.

## 3. Label-as-you-go exemplar loop

The durable lever against the synthetic-sprite domain gap, built end-to-end.

- **Persistence.** `DetectionLabel` added to `shared/types.ts`; the FROZEN `ipc.ts` widened
  additively (`labels.{load,append,clear}`); `detection-labels.json` handlers in `persistence.ts`;
  preload wrappers; `store/labels.ts` hydrated on boot in `App.tsx`. Each label stores
  `{embedding, cropPng, speciesId, model, preprocVersion, regulation, createdAt, wasAutoTop1}` —
  **the crop image is kept** so exemplars survive a model/preproc change (re-embeddable), unlike a
  bare embedding.
- **Capture.** `labelCapture.ts::captureLabel` segments+embeds a raw crop on the production path;
  Detection's `handleOverride` fires it on every slot confirm/correct that has a crop
  (screenshot/video only). A header indicator shows "N training examples · Clear".
- **Augmentation.** `exemplars.ts::augmentTableWithLabels` appends parity-matching label embeddings
  as extra reference entries (name + `baseSpeciesId` via `gen`); Detection memoizes the augmented
  table and passes it to `detectOpponentTeam`. `matchEmbedding` dedupes best-per-species (max-cosine
  1-NN) so multi-exemplar species surface once. **Self-improving locally** — no rebuild.

Files: `shared/types.ts`, `shared/ipc.ts`, `main/ipc/persistence.ts`, `preload.ts`, `App.tsx`,
`store/labels.ts`, `labelCapture.ts`, `exemplars.ts`, `iconMatcher.ts`; `exemplars.test.ts`,
`boxEmbeddings.test.ts`.
